# Membrane — Interactive Hero Section

## Summary

A full-viewport dark section with a living mesh surface rendered on a `<canvas>`. The mesh is a 50×30 grid of vertices connected by spring physics. Cursor proximity deforms the surface — vertices push outward and waves ripple through neighbors. Click for a stronger impulse. On top of the canvas, centered content (badge, title, subtitle, buttons, social proof) loads with a cinematic staggered entrance. When the cursor is over the section, dots beneath the content area dim to preserve readability.

---

## Visual Specification

### Background & Canvas

- Section: `100vh`, `100vw`, `overflow: hidden`, `background: #08080a`
- Canvas: absolute, fills entire section, renders at `devicePixelRatio` scale
- Canvas fades in on load: `opacity 0 → 1` over 2 seconds, `power2.inOut` easing (GSAP)

### Mesh Grid

- **Dimensions**: 50 columns × 30 rows
- **Lines**: both horizontal and vertical, connecting all vertices. Stroke: `rgba(255,255,255,0.018)`, line width `0.5px`
- **Vertices**: rendered as filled circles. Each vertex has:
  - A rest position on the grid
  - Current displaced position (spring physics)
  - An `offsetZ` value simulating depth (used for vertical screen displacement: `screenY = y - offsetZ * 0.3`)

### Vertex Color System

Colors vary based on distance from the viewport center:

- **Edge vertices** (far from center): cool grey `rgb(130, 130, 140)`
- **Center vertices** (near center): warm ivory `rgb(245, 235, 210)`
- **Active vertices** (displacement > 3): bright warm white `rgb(255, 252, 245)`

The blending uses a `centerFactor` (0 at edges, 1 at center):
```
centerFactor = pow(1 - min(1, distFromCenter / maxDist * 1.4), 1.5)
```

Edge-to-center interpolation:
```
r = 130 + (245 - 130) * centerFactor
g = 130 + (235 - 130) * centerFactor
b = 140 + (210 - 140) * centerFactor
```

When displacement > 3, blend further toward `(255, 252, 245)` proportional to displacement.

### Vertex Size

```
size = 1 + centerFactor * 1 + (displacement > 0.5 ? displacement * 0.05 : 0)
```

Base size ~1px at edges, ~2px at center, grows slightly when displaced.

### Vertex Alpha

```
baseAlpha = 0.06 + centerFactor * 0.18
activeAlpha = displacement > 0.5 ? min(0.9, displacement * 0.05) : 0
alpha = max(baseAlpha, activeAlpha)
```

### Vertex Glow

Vertices with `displacement > 1.5` or `centerFactor > 0.4` get an additional glow circle at `size + 6` radius with reduced alpha.

### Content Zone Dimming

When the cursor is inside the section, vertices within an elliptical zone centered on the viewport (`zoneW = width * 0.35`, `zoneH = height * 0.35`) are dimmed:

```
ellipseDist = ((restX - cx) / zoneW)² + ((restY - cy) / zoneH)²
if ellipseDist < 1:
  dim = (1 - ellipseDist) * 0.7
  alpha *= (1 - dim)
```

This creates a soft readable area around the content that fades smoothly to normal brightness at the edges.

## Physics System

### Constants

| Parameter | Value  | Purpose                          |
|-----------|--------|----------------------------------|
| SPRING    | 0.025  | Spring stiffness back to rest    |
| DAMPING   | 0.93   | Velocity decay per frame         |
| SPREAD    | 0.18   | Wave propagation between neighbors |

### Per-Vertex State

```
x, y          — current position
restX, restY  — rest position on grid
vx, vy        — velocity in X/Y
offsetZ       — simulated depth displacement
velZ          — velocity in Z
```

### Cursor Interaction

- **Hover radius**: 130px, force strength: 15
- **Click (pressing) radius**: 200px, force strength: 30
- Force direction: pushes vertex away from cursor position
- Force on Z: `force * 0.4` (upward push)
- Force on X/Y: `(dx/dist) * force * 0.12`

When cursor leaves the section, mouse position resets to `(-9999, -9999)` — no vertices are affected.

### Spring Return

Each frame:
```
vx += (restX - x) * SPRING
vy += (restY - y) * SPRING
velZ -= offsetZ * SPRING * 1.5
```

### Damping

```
vx *= DAMPING (0.93)
vy *= DAMPING
velZ *= DAMPING
```

### Wave Propagation

After all vertices update, each vertex's `velZ` is influenced by its 4 direct neighbors (up, down, left, right):
```
velZ += (neighbor.offsetZ - this.offsetZ) * SPREAD
```

This creates realistic ripple waves that travel outward from the point of interaction.

### Breathing Animation

Continuous ambient motion applied every frame before physics:
```
breathe = sin(time * 0.6 + col * 0.15 + row * 0.12) * 2
velZ += (breathe - offsetZ) * 0.002
```

Very gentle — keeps the surface alive when no cursor interaction is happening.

## Content Layer

All content is centered vertically and horizontally over the canvas, `pointer-events: none` on the container (buttons opt back in with `pointer-events: auto`).

### Entrance Animation (GSAP)

All elements marked with `data-reveal` attribute animate in sequence:

- **Initial state**: `opacity: 0`, `y: 30px`, `scale: 0.94`, `filter: blur(6px)`
- **Final state**: `opacity: 1`, `y: 0`, `scale: 1`, `filter: blur(0px)`
- **Duration**: 1 second per element
- **Easing**: `power4.out`
- **Stagger**: 130ms between elements
- **Delay**: 500ms before sequence starts

Order of reveal: badge → title → subtitle → buttons → social proof

### Award Badge

Horizontal layout: left laurel SVG + center text + right laurel SVG (mirrored).

- **Laurel SVGs**: actual decorative award symbol images (`left-award-symbol.svg`, `right-award-symbol.svg`), displayed at `36×56px` (mobile) / `44×68px` (desktop)
- **SVG treatment**: `filter: invert(1) opacity(0.4)` — inverts black SVGs to white, at 40% opacity
- **Overlap**: left SVG has `margin-right: -15px`, right SVG has `margin-left: -15px` — they overlap into the text area
- **Center text**: "#1 Interactive Section" in Playfair Display, `11px` mobile / `13px` desktop, `font-bold`, `tracking-wide`, `text-white/55`
- **Stars**: 5 star SVGs below the text, each 13×13px, `fill: rgba(255,255,255,0.3)`. Star polygon: `points="12,2 15.09,8.26 22,9.27 17,14.14 18.18,21.02 12,17.77 5.82,21.02 7,14.14 2,9.27 8.91,8.26"`

### Title

- **Font**: Playfair Display (`var(--font-playfair)`), `font-weight: 500`
- **Size**: `clamp(3.2rem, 8vw, 6.5rem)` — responsive, large
- **Line-height**: `1` (tight)
- **Letter-spacing**: `-0.02em`
- **Color**: `white`
- **Text**: "Touch the invisible"
- **Highlighted word** ("invisible"): `text-white/40` by default (dimmed), transitions to `text-white` on hover over 500ms. No underline, no gradient — the dimming itself is the design.
- **Spacing**: `margin-top: 28px` from badge

### Subtitle

## Technical Requirements

- **Canvas**: HTML5 Canvas 2D, rendered at device pixel ratio
- **Animation library**: GSAP (for canvas fade-in and content entrance only — mesh physics are pure requestAnimationFrame)
- **Framework**: React (useEffect + useRef pattern), but the logic is portable to vanilla JS
- **Performance**: ~1500 vertices (50×30), each with 4-neighbor spring propagation. Runs comfortably at 60fps on modern hardware
- **Event listeners**: mousemove, mouseleave, mousedown, mouseup on the container element
- **Cleanup**: all animation frames cancelled, all event listeners removed on unmount
