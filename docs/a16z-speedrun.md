# Dudilig — a16z Speedrun Application Notes

**One-liner.** The Compliance OS for tokenized real-world assets.

**What we ship.** Continuous KYC/AML monitoring, smart-contract attestation (verifying on-chain code matches off-chain claims), regulatory filing tracking, and investor-registry management — turning compliance from a launch-blocker into a continuously verifiable trust signal.

**Live product.** [dudilig.com](https://dudilig.com) · [github.com/hershmalik/dudilig](https://github.com/hershmalik/dudilig)

---

## What changed in the last 24 hours

We shipped the public **Trust Certificate** — the network-effect lever from our PRD.

Before the sprint, our Claude Opus 4.5 contract analyzer worked, but every output lived behind our app shell. An issuer could see the verdict; nobody else could. That's a closed loop.

Now every analysis can be **published to a public, signed URL** that anyone — allocators, exchanges, regulators, journalists — can hit without an account. Each certificate is:

1. **Generated by the production analyzer.** Same Claude Opus 4.5 model, same rule set, same JSON schema. Not a separate path, not a mock.
2. **Cryptographically signed.** SHA-256 over a deterministic canonical-JSON serialization of the full attestation payload (issuer, token, standard, claimed facts, contract source, model output, timestamp). Anyone with the same inputs can recompute the hash byte-for-byte.
3. **Embeddable.** A lightweight `/embed` route renders a transparent-background card that drops into any data room, fund page, or pitch deck via iframe.
4. **Shareable.** OG images render inline previews on Twitter/X, LinkedIn, and Telegram.
5. **Fully public.** No auth, no app chrome, no "talk to sales."

The mismatch case is the wedge. This is the demo we'll lead with: an issuer who tells LPs one thing in the offering documents and ships a contract that says something else. Today, no one notices until somebody gets sued. Dudilig catches it in 18 seconds and publishes the receipt.

## Why this matters for Speedrun

The PRD pitches Dudilig as the work platform for RWA compliance. The hard part isn't the AI — it's getting the network on it. The Trust Certificate is the unlock:

- **For issuers:** a credibility instrument they can put in their data room *today*, with no integration work on the LP side.
- **For allocators / exchanges / market makers:** a way to gate counterparty risk on something better than a PDF and a handshake.
- **For Dudilig:** every published certificate is a public link back to us, attached to a tokenized asset that already has distribution. That's the loop.

## What's actually running

- **Stack:** Next.js 16.2.4, React 19, Tailwind 4, Anthropic SDK (`claude-opus-4-5`)
- **Production:** [dudilig.com](https://dudilig.com) on Render
- **Sprint env:** Replit Agent 4 / Replit Workspace
- **Persistence:** file-based JSON under `data/certificates/{id}.json` (24-hour sprint constraint; production migrates to Postgres)
- **Hash:** SHA-256 over canonical JSON via `lib/crypto/hash.ts`
- **No mocks on the critical path:** every certificate is the literal output of a real Claude call against real Solidity.

## What we cut (and why we're fine cutting it)

- **EAS testnet anchoring.** Stretch goal. The hash is real and reproducible today; anchoring it to Sepolia adds notarization but doesn't change the user story. Two-day follow-up.
- **Continuous re-attestation cron.** Already designed; one-week build. Doesn't need to ship to make the demo land.
- **Form D drafting agent.** Separate workstream from the trust loop.

## What we'd build next with funding

1. **EAS / on-chain anchor** — every certificate hash committed to Base/Sepolia; `verifiedOn` field on the public page.
2. **Watcher cron** — re-attest every deployed contract daily; alert issuer + invalidate certificate on drift.
3. **Form D / Form ATS / MiCA whitepaper drafting agents** — same Claude pipeline, different output shape, into FINRA / SEC / ESMA filing flows.
4. **Issuer registry handshake** — LP wallet signs once, sees real-time compliance status of every token they hold.
